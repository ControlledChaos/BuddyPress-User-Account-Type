Installation

Installation is simple. just download buddypress-user-account-type-pro.zip from you wp.bp.shop profile after purchasing. Upload it to your site's wp-content/plugins directory, unzip it and activate it from wp-admin -> plugins.

Or, go to up-admin -> plugins -> add new -> upload and then upload buddypress-user-account-type-pro.zip file. after uploading it,activate it. and you are ready to have its power!

Important Note: You must need BuddyPress installed and activated to use this plugin

Configuring basic settings

Step 1: If you want to allow different type of users, first you need to create type names. BuddyPress User Account Type PRO searches for if theres any drop down select box field in 'base' BuddyPress profile group. So, you must create a drop down select box field to define user type and add type names.

Step 2: Now go to wp-admin -> BUAT PRO settings, you will get all select box type fields, you need to select one which will define user types. it should look like,

Step 3: After selected type field, you will get all basic setting fields, like: Option [ Select default type ]: It will be default user type, if user select nothing during registration, or, if any users type is not assigned while trigging existing users, default type will be assigned.
Option [ Select page to show (type name 1) type users ]: This is the page name where (type name 1) type user directory should show,
Teachers directory is example here
Option [ Select role for (type name 1) type users ]: Select Role for (type name 1) type users, teachers type is example here. Option [ Whats about existing users? ]: What you want to do existing users? if you select Do Nothing it will just do noting, if you select Assign type according to role, it will assign type value to existing users according to Roles and of course which are not selected their types, you may choose which type should assign for which role, after selecting Assign type according to role, you'll get option to select that, it should look like: if you select Assign default type to all, it will just assign default type to existing users who are not selected own type, but it wont change any role and capability.

Important note: This option will appear only first time you configure BuddyPress User Account Type PRO. But you can have role triggering option once you configured it.

Option [ Exclude IDs from change role ( comma separated ) ]: you may choose some user IDs not to change type or roles By default, admins are excluded from triggering types and roles.
Option [ Users to display per page ]: BuddyPress shows 10 members per page in members directory, you may increase or decrease it, it will work both for BuddyPress members directory and different user type directories.
Option [ Can user change type after registration? ]: You can allow users to change own type after registration from their BuddyPress profile. if you don't allow users to change their own profile, user can't do that but admin can change it by editing his/her profile.

Important note: Thats all for basic setting, You must click "save change" before going other settings.

Configuring profile data settings

Step 4: You may manage different fields for different types of users from profile data settings section. but before configuring this section, you must configure basic settings section. here is how it should look:
Option [ Exclude type selection users ]: You may decide to have some protected account type that is not open to users during registration. Which means, user cant select these protected type as their type, but only admin can assign these type to users manually editing user's profile. If you want that, selected types from here which you want as protected.

Option [ Exclude fields for (user type 1) type users ]: select which fields you don't want to show for user type 1 users. user cant show/edit excluded fields by own, but admin can edit it.

Option [ Include fields for (user type 1) type users at registration form ]: After excluding fields from different type users save it, and there will be generate a section to include fields at registration form. you may select type associated fields to show in registration form. so, when user select own type during registration, associated fields of that type will be appeared via ajax request. which I like to call, Conditional Profile Fields

Option [ Select fields to show in profile loop ]: Select which fields you want to show in user directory list view. it will exclude automatically if user dosent fill up data for a field. only those fields will be visible which will contain data. and it will aligned by column automatically.

Option [ Number of fields per column ]: Select number of profile fields you want to show per colum in user directory list view. number of columns will be automatically generated depending on selected profile fields and numbers of fields you want to show per column

Configuring style settings

Step 5: You can manage style of both custom and default directories from this section, it should be look like
Option [ Default view of members directory ]: You can select default view of directories. BuddyPress default view is list view, here is another option to show it like grid view,

Option [ Allow users to change view ]: You may allow visitors to change directory view, user can select list or grid view them self.

Configuring access settings

Step 6: you can manage page/post restriction settings for different types of users
Option [ Restrict pages/posts for (user type 1) type users ]: you can place here comma separated post/page IDs so, this particular type of users cant access these pages.

Option [ Restrict URL for (user type 1) type users ]: You may want to restrict specific URL from specific user types. Use only slug here, Example: user /example-page if you want to restrict yoursite.com/example-page For BuddyPress varialble URLs ( members, Groups ), user [user_name] where you need to place displayed users user name, and place [group_name] where you need current group name.
Example: If you want to restrict users profile, user /members/[user_name]/profile/public
If you want to restrict users profile, user /members/[user_name]/profile/public
Important Note: If you want to restrict BuddyPress component root urls,e.g: members, groups, activity, User URL restriction instead of ID restriction, so, Sub pages will not restricted.

Option [ Select page to show Restriction notice for (user type 1) type users ]: You may select a page to show notice/message/reason of the restriction. Plugin will redirect users while they will try to access restricted pages.

Option [ Text/html restriction ]: You may select a part of content only to be visible for particular user type users. syntex is:
[buatp user_type="user_type_1,user_type_2"]
YOUR CONTENTS HERE
[buatp]
which means, this particular text content is only for user_type_1,user_type_2 type user. and output of this:
Option [ Restriction message for shortcode restriction ]: You can add custom notification text for content restriction

Shortcode reference

[buatp]: using [buatp] shrotcode, you can have all members inside any page/post
[buatp] filters:

  • user_type: can be a specific user type or comma separated user types.
    example: [buatp user_type="teacher"]
  • exclude: can be exclude users by comma separated ID, will exclude by force if user exist in given type
    example: [buatp user_type="teacher" exclude="2"]
  • arrange_by: user can me arranged by active, newest, popular, online, alphabetical, random
    example: [buatp user_type="teacher" arrange_by="popular"]
  • max: maximum numbers of users to show
    example: [buatp user_type="teacher" arrange_by="active" max="2"]
